# Jenkins Allure Report Setup - Quick Reference

## Prerequisites Checklist

### Jenkins Server Requirements
- [ ] Node.js (v14+) installed
- [ ] npm installed
- [ ] Git installed

### Jenkins Plugins Required
- [ ] Pipeline Plugin
- [ ] Allure Plugin (Apache - org.jvnet.hudson.plugins:allure-jenkins-plugin)
- [ ] Git Plugin

## 1-2-3 Setup Guide

### Step 1: Install Allure Plugin on Jenkins
```
Manage Jenkins → Manage Plugins → Available Tab
Search: "Allure" 
Install: "Allure Plugin" (official Apache)
Restart Jenkins
```

### Step 2: Create Jenkins Pipeline Job
```
1. Click "New Item"
2. Name: "Playwright-Tests"
3. Select: "Pipeline"
4. Click "OK"
```

### Step 3: Configure Pipeline
```
Definition: Pipeline script from SCM
SCM: Git
Repository URL: https://github.com/YOUR_USERNAME/Playwright-GoogleAutomate.git
Branch: */main
Script Path: Jenkinsfile
Save & Build
```

## Common Issues & Solutions

| Issue | Solution |
|-------|----------|
| No Allure Report link on job page | Install Allure Plugin - Manage Jenkins → Manage Plugins |
| Blank report | Pull latest code with fixed Jenkinsfile |
| npm: command not found | Add Node.js to Jenkins agent or install npm |
| Test results missing | Check Console Output for test execution errors |
| Permission denied on workspace | Check Jenkins user has workspace access |

## Verify Everything Works

### Local Verification
```bash
npm install
npm test
npm run allure:generate
npm run allure:open
```

### Jenkins Verification
1. Trigger build: "Build Now"
2. Wait for completion
3. Click "Allure Report" link (left sidebar)
4. See populated report with tests, steps, and statistics

## File Structure

```
Repository
├── Jenkinsfile          ← Pipeline configuration
├── playwright.config.js ← Test configuration with Allure reporter
├── tests/
│   └── login.spec.js    ← Tests with detailed steps
├── pages/               ← Page Object Models
├── test-data/           ← Test data
├── allure-results/      ← Generated by tests (not committed)
└── allure-report/       ← Generated by npm script (not committed)
```


## Allure Report Features

### Dashboard
- Total test count
- Pass/Fail ratio
- Test duration
- Trend graphs across builds

### Tests Tab
- Individual test cases
- Detailed execution steps
- Screenshots/Videos on failure
- Execution duration

### Timeline
- Test execution order
- Parallel execution info (if enabled)

### Behaviors
- Tests grouped by feature
- Better organization for large suites

## Environment Details in Report

Reports now include:
- Browser information
- Test execution time
- OS/Platform details
- Test annotations (user, description, expected result)

## Build History

Each Jenkins build preserves:
- Test results (allure-results)
- HTML report (playwright-report)
- JUnit XML report (test-results/junit.xml)
- Allure test artifacts

Access previous builds: Job → Build History → Click build number → Allure Report

## Notifications (Optional)

### Email
Add to Jenkinsfile post section:
```groovy
emailext(subject: 'Test Report', body: 'See: ${BUILD_URL}allure/', to: 'team@company.com')
```

### Slack
```groovy
slackSend(message: "${BUILD_URL}allure/", color: currentBuild.result == 'SUCCESS' ? 'good' : 'danger')
```
